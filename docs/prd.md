# Tab Master - Chrome New Tab 插件 PRD

## 1. 产品概述

**产品名称**: Tab Master  
**类型**: Chrome New Tab 插件  
**一句话描述**: 替代Chrome默认新标签页，提供强大的Tab搜索、分组管理和快捷访问功能。  
**目标用户**: 同时打开30-100个Tab，需要高效管理浏览器会话的用户

---

## 2. 功能需求

### 2.1 核心功能

#### P0 - 必须实现

| 功能 | 描述 |
|------|------|
| **Tab列表展示** | 显示所有已打开Tab，按分组折叠展示，每个卡片显示：标题 + favicon + 域名 |
| **关键词搜索** | 顶部搜索框，支持实时过滤，匹配Tab标题和URL |
| **自动域名分组** | 同一域名的Tab自动归为一个分组，显示网站名称和Tab数量 |
| **手动分组** | 用户可创建自定义分组，手动拖拽Tab进入分组，支持嵌套最多2层 |
| **切换Tab** | 点击Tab卡片，立即跳转到该页面 |
| **关闭Tab** | 每个卡片有关闭按钮，点击直接关闭该Tab |
| **分组操作** | 分组支持：展开/折叠、重命名、删除（Tab保留）、合并 |
| **最近关闭** | 底部显示最近关闭的20个Tab，仅保留当天，点击可恢复 |

#### P1 - 增强功能

| 功能 | 描述 |
|------|------|
| **最常访问** | 统计访问频率，显示Top 10最常访问的Tab |
| **新建Tab** | 快捷按钮，点击打开空白新Tab |
| **多窗口支持** | 显示所有窗口的Tab，可在配置中切换"仅当前窗口" |

### 2.2 用户交互流程

```
用户打开新标签页
    ↓
显示全量Tab列表（按分组折叠）
    ↓
用户可通过以下方式操作：
    ├── 输入关键词 → 实时搜索过滤
    ├── 点击分组 → 展开/折叠
    ├── 点击Tab卡片 → 跳转到该页面
    ├── 点击关闭按钮 → 关闭该Tab
    ├── 拖拽Tab → 移动到其他分组
    ├── 右键Tab → 更多操作菜单
    ├── 新建分组 → 创建自定义分组
    ├── 底部查看最近关闭 → 点击恢复
    └── 底部查看最常访问 → 点击跳转
```

### 2.3 数据存储

| 数据类型 | 存储方式 | 说明 |
|----------|----------|------|
| 用户分组结构 | Chrome Storage API | 包含分组ID、名称、层级关系 |
| 最近关闭列表 | Chrome Storage API | 最多20条，仅保留当天 |
| 最常访问 | Chrome Storage API | 最多Top 10，每天凌晨重置计数 |
| 用户设置 | Chrome Storage API | 如"仅当前窗口"等偏好 |

## 3. 验收标准 (AC)

### AC-1: Tab列表展示
```
Given: 用户打开新标签页
When:  插件加载完成
Then:  显示所有已打开Tab，按域名自动分组，分组默认折叠
      每个Tab卡片显示：favicon + 标题 + 域名
```

### AC-2: 关键词搜索
```
Given: 用户在搜索框输入关键词
When:  输入过程中（防抖300ms）
Then:  实时过滤显示匹配的Tab，匹配范围为标题和URL
      无匹配时显示空态提示"没有找到匹配的Tab"
```

### AC-3: Tab切换
```
Given: 用户点击某个Tab卡片
When:  点击事件触发
Then:  立即跳转到该Tab页面，激活该窗口
```

### AC-4: Tab关闭
```
Given: 用户点击Tab卡片的关闭按钮
When:  点击事件触发
Then:  立即关闭该Tab，卡片从列表移除
      同时将该Tab添加到"最近关闭"列表
```

### AC-5: 手动分组
```
Given: 用户拖拽Tab到另一分组
When:  拖拽释放
Then:  Tab移动到目标分组，刷新分组Tab列表
      分组结构保存到Chrome Storage
```

### AC-6: 最近关闭
```
Given: 用户关闭了某个Tab
When:  关闭事件触发
Then:  该Tab保存到最近关闭列表（最多20条）
      每天0点清空前一天的数据
      点击最近关闭的Tab可恢复
```

### AC-7: 分组操作
```
Given: 用户在分组标题上右键
When:  右键菜单触发
Then:  显示菜单：展开/折叠 | 重命名 | 删除分组 | 合并到...
```

## 4. 非功能需求

### 4.1 性能要求
- 页面加载时间 < 500ms
- 搜索响应时间 < 100ms
- 支持最多500个Tab不卡顿

### 4.2 兼容性
- Chrome 88+（支持Manifest V3）
- 响应式布局，适配不同窗口大小

### 4.3 技术栈
- Manifest V3
- React 18
- TailwindCSS
- Chrome Extension API (chrome.tabs, chrome.storage)

---

## 5. 优先级说明

| 优先级 | 说明 |
|--------|------|
| P0 | 核心流程，插件价值所在 |
| P1 | 增强体验，不影响核心功能 |
| P2 | 锦上添花，后续迭代考虑 |

## 6. 不在本次范围

- Tab页签之间的拖拽排序（仅支持跨分组移动）
- Tab会话保存/恢复（保存所有Tab状态到文件）
- 云端同步
- 多设备同步
