# decisions.md - Tab Master 关键决策记录

## 2026-05-04 项目初始化决策

### Q1: 插件核心用途
- **决策**: 综合功能（A+B+C+D）
- **原因**: 用户需要Tab搜索、分组管理、最近关闭、最常访问全部功能

### Q2: Tab数量规模
- **决策**: 30-100个Tab量级
- **原因**: 中度使用，需要分组管理

### Q3: 分组方式
- **决策**: 自动+手动组合分组
- **原因**: 按域名自动聚合，减少手动操作；同时支持手动调整

### Q4: 视觉风格
- **决策**: 卡片式设计，每个Tab有缩略图预览
- **原因**: 用户偏好卡片式展示

### Q5: 具体实现细节（老王建议）
- **卡片内容**: 仅显示标题 + favicon + 域名（不包含页面截图，性能考虑）
- **搜索范围**: 标题 + URL 都要匹配
- **分组层级**: 最多2层嵌套
- **最近关闭**: 保存20条，仅保留当天
- **多窗口**: 支持管理所有窗口的Tab
- **侧边栏改为底部工具栏**: New Tab页面高度有限，底部更高效
- **分组默认折叠**: 避免大量Tab同时展示造成混乱

### 技术栈选择
- **决策**: Manifest V3 + React + TailwindCSS
- **原因**: 
  - Manifest V3是Chrome扩展的未来趋势
  - React适合复杂交互界面
  - TailwindCSS快速样式开发

### 不包含的功能
- Tab截图预览（性能开销大）
- 云端同步
- 多设备同步
- 会话保存/恢复
